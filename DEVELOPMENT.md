# 开发文档

## 架构设计

### 子进程架构

本项目采用主进程+子进程的架构设计：

- **主进程**：负责UI界面、配置管理、进程管理
- **子进程**（映射子进程.py）：负责手柄输入监听和映射执行

**优点**：
- 映射功能崩溃不会影响主程序
- 可以独立关闭映射功能而不退出程序
- 便于进程管理和监控

**通信方式**：
- 主进程通过stdin向子进程发送JSON命令
- 子进程通过stdout向主进程发送事件和日志

### 核心模块

#### 主程序.py

**App类** - 主应用程序类
- 继承自 `tk.Tk`
- 负责UI界面和事件处理
- 管理子进程生命周期

**关键方法**：

| 方法名 | 功能 |
|--------|------|
| `start_mapper_subprocess` | 启动映射子进程 |
| `stop_mapper_subprocess` | 停止映射子进程 |
| `send_command_to_subprocess` | 向子进程发送命令 |
| `subprocess_output_listener` | 监听子进程输出 |
| `refresh_process_list` | 刷新进程列表（异步） |
| `on_process_list_click` | 处理进程列表点击事件 |

#### 映射子进程.py

**MapperSubprocess类** - 映射子进程主类
- 独立运行，通过stdin/stdout与主进程通信
- 负责手柄输入监听
- 执行映射操作（鼠标点击、键盘按键等）

**关键方法**：

| 方法名 | 功能 |
|--------|------|
| `command_listener` | 监听主进程命令 |
| `gamepad_listener` | 手柄输入监听循环 |
| `execute_button_action` | 执行按钮映射动作 |
| `execute_touch` | 执行触摸/点击操作 |

**支持的命令**：
- `start_mapping` - 启动映射
- `stop_mapping` - 停止映射
- `update_config` - 更新配置
- `exit` - 退出子进程

#### window_locator.py

窗口定位工具，用于识别和定位目标窗口。

#### 虚拟键盘网格布局.py

虚拟键盘界面，支持单键和组合键选择。

## 输入处理流程

### 手柄输入处理

1. 子进程的 `gamepad_listener` 线程持续监听手柄事件
2. 解析事件类型（按钮、摇杆、扳机）
3. 根据配置执行相应操作：
   - 按钮按下 → 触发点击或按键
   - 摇杆移动 → 控制鼠标或触发方向键
   - 扳机按下 → 根据阈值触发操作
4. 通过stdout向主进程发送事件通知（用于UI高亮）

### 坐标系统

**屏幕坐标**：
- 相对于整个屏幕的绝对坐标
- 原点 (0,0) 在屏幕左上角

**窗口坐标**：
- 相对于目标窗口客户区的相对坐标
- 窗口移动时坐标自动跟随

## 进程管理

### 进程检测

- 通过 `psutil` 库遍历所有进程
- 检测关键词：主程序、gamepad、mapper、手柄、映射
- 区分子进程和主程序

### 进程关闭

- 主程序：需要额外确认（可能导致配置丢失）
- 子进程：可直接关闭（映射功能停止）
- 当前程序的子进程：关闭后自动更新UI状态

## 开发规范

### 代码风格
- 使用4空格缩进
- 函数名使用小写+下划线
- 类名使用驼峰命名法
- 文件头添加功能说明文档字符串

### 注释规范
- 模块开头添加功能说明
- 复杂逻辑添加简要注释
- 避免冗余注释和空行

## 依赖库

| 库名 | 用途 | 版本 |

```bash
pip install -r requirements.txt
```

## 打包发布

### 使用PyInstaller打包

```bash
pyinstaller --onefile --windowed --name "游戏手柄触屏映射工具" 主程序.py
```

### 打包包含子进程

需要确保 `映射子进程.py` 在同一目录下，主程序会动态调用。
